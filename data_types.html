Primitive Types

C provides several basic data types that are built into the language. These are often called primitive types.

Integer Types

Integer types store whole numbers without fractional parts.

Type Size (typical) Range (typical) Format Specifier
char 1 byte -128 to 127 or 0 to 255 %c (character)
%d (integer)
unsigned char 1 byte 0 to 255 %c (character)
%u (integer)
short or short int 2 bytes -32,768 to 32,767 %hd
unsigned short 2 bytes 0 to 65,535 %hu
int 4 bytes -2,147,483,648 to 2,147,483,647 %d or %i
unsigned int 4 bytes 0 to 4,294,967,295 %u
long or long int 4 or 8 bytes -2,147,483,648 to 2,147,483,647 (4 bytes)
or -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807 (8 bytes)		 %ld
unsigned long 4 or 8 bytes 0 to 4,294,967,295 (4 bytes)
or 0 to 18,446,744,073,709,551,615 (8 bytes)		 %lu
long long 8 bytes -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807 %lld
unsigned long long 8 bytes 0 to 18,446,744,073,709,551,615 %llu

The exact size and range of integer types can vary depending on the compiler and platform. The values shown above are typical for many modern systems.

int count = 42;
unsigned int positiveCount = 100;
long long bigNumber = 9223372036854775807LL;  // Note the LL suffix for long long literals

printf("Count: %d\n", count);
printf("Positive count: %u\n", positiveCount);
printf("Big number: %lld\n", bigNumber);

To ensure portability, you can use types from <stdint.h> (C99 and later) that have fixed sizes, such as int32_t for a 32-bit integer.

Floating-Point Types

Floating-point types store numbers with fractional parts.

Type Size Precision Format Specifier
float 4 bytes ~7 decimal digits %f
double 8 bytes ~15 decimal digits %lf or %f
long double 12 or 16 bytes ~19 decimal digits %Lf 
float pi = 3.14159f;  // Note the f suffix for float literals
double precisePI = 3.14159265358979323846;
long double veryPrecisePI = 3.14159265358979323846L;  // Note the L suffix for long double literals

printf("Pi (float): %.5f\n", pi);
printf("Pi (double): %.15f\n", precisePI);
printf("Pi (long double): %.18Lf\n", veryPrecisePI);

Floating-point arithmetic can lead to precision errors due to the way these numbers are represented in binary. Never use floating-point types for exact calculations like currency.

Character Type

The char type is used to store single characters, but it's also an integer type.

char letter = 'A';  // Character literal in single quotes
char newline = '\n';  // Escape sequence for newline

printf("Letter: %c\n", letter);
printf("ASCII value of letter: %d\n", letter);  // Prints 65 for 'A'

Common Escape Sequences

  • \n - Newline
  • \t - Tab
  • \r - Carriage return
  • \b - Backspace
  • \f - Form feed
  • \v - Vertical tab
  • \\ - Backslash
  • \' - Single quote
  • \" - Double quote
  • \0 - Null character
  • \xhh - Hexadecimal value (e.g., \x41 for 'A')

Boolean Type (C99 and later)

C99 introduced a boolean type through the <stdbool.h> header.

#include <stdbool.h>

bool isValid = true;
bool isComplete = false;

if (isValid) {
    printf("The value is valid.\n");
}

Before C99, boolean values were typically represented using integers, with 0 for false and non-zero values for true.

Void Type

The void type represents the absence of a type. It's commonly used for functions that don't return a value and for generic pointers.

void printMessage() {
    printf("This function doesn't return a value.\n");
}

void* genericPointer(void* data) {
    // Can point to any data type
    return data;
}

Variables

Variables are named storage locations that hold values of specific data types. They are the fundamental building blocks for storing and manipulating data in C programs.

Variable Declaration and Initialization

In C, variables must be declared before they can be used. Declaration specifies the data type and name of the variable.

// Variable declaration
int count;           // Declares an integer variable named 'count'
float temperature;   // Declares a float variable named 'temperature'

// Variable initialization
count = 10;          // Assigns the value 10 to 'count'
temperature = 98.6f;  // Assigns the value 98.6 to 'temperature'

// Declaration and initialization in a single statement
int age = 25;        // Declares an integer 'age' and initializes it to 25
char grade = 'A';    // Declares a character 'grade' and initializes it to 'A'

Variable Naming Rules

When naming variables in C, follow these rules:

  • Names can contain letters, digits, and underscores
  • Names must begin with a letter or underscore
  • Names are case-sensitive (age and Age are different variables)
  • Names cannot use C keywords (like int, float, return, etc.)
  • Names should be descriptive and meaningful

Avoid using names that begin with an underscore followed by a capital letter or another underscore, as these are reserved for system use.

Variable Scope and Lifetime

The scope of a variable determines where in the program the variable can be accessed. The lifetime determines when the variable is created and destroyed.

// Global variable (file scope)
int globalVar = 100;  // Accessible throughout the file

void exampleFunction() {
    // Local variable (block scope)
    int localVar = 10;  // Only accessible within this function
    
    {
        // Nested block scope
        int nestedVar = 5;  // Only accessible within this block
        printf("Inside nested block: %d, %d, %d\n", globalVar, localVar, nestedVar);
    }
    
    // nestedVar is not accessible here
    printf("Inside function: %d, %d\n", globalVar, localVar);
}

Storage Classes

C provides several storage classes that control the scope and lifetime of variables:

  • auto - Default for local variables (automatically allocated and deallocated)
  • static - Preserves value between function calls and limits scope to the file
  • extern - Declares a variable that is defined in another file
  • register - Suggests that the variable be stored in a CPU register for faster access
// Static variable example
void countCalls() {
    static int count = 0;  // Initialized only once, value persists between calls
    count++;
    printf("This function has been called %d times\n", count);
}

Constants

Constants are variables whose values cannot be changed after initialization:

// Using const keyword
const float PI = 3.14159f;  // Value cannot be changed

// Using #define directive (preprocessor constant)
#define MAX_SIZE 100  // Symbolic constant, no memory allocation

Use const for type-safe constants and #define for simple text substitution. const variables have a specific type and scope, while #define constants are processed by the preprocessor before compilation.

Type Conversion

C allows conversion between different data types, either implicitly or explicitly.

Implicit Type Conversion (Automatic)

The compiler automatically converts one data type to another when:

  • Different data types are used in an expression
  • A value is assigned to a variable of a different type
int i = 10;
float f = 3.5f;
double d;

d = i + f;  // i is converted to float, then the result is converted to double
printf("Result: %f\n", d);  // Output: 13.500000

Implicit conversions can lead to loss of data or precision, especially when converting from a larger type to a smaller type or from floating-point to integer.

Explicit Type Conversion (Casting)

You can explicitly convert a value from one type to another using a cast operator.

float f = 3.75f;
int i;

i = (int)f;  // Explicitly cast float to int
printf("Float: %f, Integer: %d\n", f, i);  // Output: Float: 3.750000, Integer: 3

When casting from floating-point to integer, the fractional part is truncated (not rounded).

sizeof Operator

The sizeof operator returns the size in bytes of a type or variable.

int i;
float f;
double d;
char c;

printf("Size of int: %zu bytes\n", sizeof(i));
printf("Size of float: %zu bytes\n", sizeof(f));
printf("Size of double: %zu bytes\n", sizeof(d));
printf("Size of char: %zu bytes\n", sizeof(c));

The %zu format specifier is used for size_t values, which is the type returned by sizeof.
