Function Declaration and Definition

Functions are blocks of code that perform specific tasks. They help organize code, promote reusability, and make programs easier to understand and maintain.

Function Declaration (Prototype)

A function declaration tells the compiler about a function's name, return type, and parameters. It doesn't contain the function body.

// Function declaration (prototype)
int add(int a, int b);  // Note the semicolon at the end

Function Definition

A function definition provides the actual body of the function.

// Function definition
int add(int a, int b) {
    return a + b;  // Return statement
}

Complete Example

#include <stdio.h>

// Function declaration
int add(int a, int b);

int main() {
    int result = add(5, 3);
    printf("5 + 3 = %d\n", result);
    return 0;
}

// Function definition
int add(int a, int b) {
    return a + b;
}

In C, function declarations (prototypes) are typically placed in header files (.h), while function definitions are placed in source files (.c).

Parameters and Return Values

Parameters

Parameters are variables that receive values when the function is called. They are also called formal parameters.

void greet(char* name, int age) {
    printf("Hello, %s! You are %d years old.\n", name, age);
}

int main() {
    greet("Alice", 25);  // Arguments: "Alice" and 25
    return 0;
}

Return Values

Functions can return a value using the return statement. The return type is specified in the function declaration.

int square(int num) {
    return num * num;
}

double calculateArea(double radius) {
    return 3.14159 * radius * radius;
}

void printMessage() {  // No return value (void)
    printf("This function doesn't return anything.\n");
    return;  // Optional for void functions
}

If a function doesn't return a value, use the void return type. The return statement without a value is optional in void functions.

Parameter Passing

C uses "pass by value" by default, which means the function receives a copy of the argument's value, not the original variable.

Pass by Value

void increment(int num) {
    num++;  // Modifies the local copy, not the original variable
    printf("Inside function: %d\n", num);
}

int main() {
    int x = 5;
    printf("Before function call: %d\n", x);
    increment(x);
    printf("After function call: %d\n", x);  // Still 5, unchanged
    return 0;
}

Pass by Reference (using pointers)

To modify the original variable, you can pass its address (using pointers).

void incrementByReference(int* numPtr) {
    (*numPtr)++;  // Modifies the value at the address pointed to by numPtr
    printf("Inside function: %d\n", *numPtr);
}

int main() {
    int x = 5;
    printf("Before function call: %d\n", x);
    incrementByReference(&x);  // Pass the address of x
    printf("After function call: %d\n", x);  // Now 6, modified
    return 0;
}

When passing pointers, be careful to ensure they point to valid memory locations to avoid undefined behavior.

Function Prototypes

Function prototypes inform the compiler about the function's signature before its actual definition.

#include <stdio.h>

// Function prototypes
double calculateArea(double radius);
void printResult(double area);

int main() {
    double radius = 5.0;
    double area = calculateArea(radius);
    printResult(area);
    return 0;
}

double calculateArea(double radius) {
    return 3.14159 * radius * radius;
}

void printResult(double area) {
    printf("The area is: %.2f\n", area);
}

Function prototypes are essential when functions are defined after they are called or in different source files.

K&R Style vs. ANSI C Style

In older K&R style C, function declarations didn't include parameter types. Modern ANSI C requires parameter types in both declarations and definitions.

// K&R style (old, not recommended)
int add(); // No parameter information

// ANSI C style (modern, recommended)
int add(int a, int b); // With parameter types

Recursion

Recursion occurs when a function calls itself. It's useful for solving problems that can be broken down into smaller, similar subproblems.

Factorial Example

int factorial(int n) {
    if (n <= 1) {  // Base case
        return 1;
    } else {  // Recursive case
        return n * factorial(n - 1);
    }
}

int main() {
    printf("5! = %d\n", factorial(5));  // Output: 5! = 120
    return 0;
}

Fibonacci Example

int fibonacci(int n) {
    if (n <= 1) {  // Base cases
        return n;
    } else {  // Recursive case
        return fibonacci(n - 1) + fibonacci(n - 2);
    }
}

int main() {
    printf("Fibonacci(7) = %d\n", fibonacci(7));  // Output: Fibonacci(7) = 13
    return 0;
}

Recursive functions can lead to stack overflow if they call themselves too many times without reaching a base case. Always ensure there's a proper termination condition.

Many recursive algorithms can be rewritten iteratively for better performance, especially when the recursion depth might be large.

Function Pointers

Function pointers allow you to store and pass functions as arguments to other functions, enabling more flexible and dynamic code.

Declaring and Using Function Pointers

int add(int a, int b) {
    return a + b;
}

int subtract(int a, int b) {
    return a - b;
}

int main() {
    // Declare a function pointer
    int (*operation)(int, int);
    
    // Assign the address of add to the function pointer
    operation = &add;  // The & is optional
    printf("Result of add: %d\n", operation(5, 3));  // Output: 8
    
    // Assign the address of subtract to the function pointer
    operation = subtract;  // Without &
    printf("Result of subtract: %d\n", (*operation)(5, 3));  // Output: 2
    
    return 0;
}

Function Pointers as Arguments

int apply(int (*func)(int, int), int a, int b) {
    return func(a, b);
}

int multiply(int a, int b) {
    return a * b;
}

int main() {
    printf("Result of add: %d\n", apply(add, 5, 3));  // Output: 8
    printf("Result of multiply: %d\n", apply(multiply, 5, 3));  // Output: 15
    return 0;
}

Array of Function Pointers

int main() {
    // Array of function pointers
    int (*operations[3])(int, int) = {add, subtract, multiply};
    
    int a = 10, b = 5;
    char* opNames[3] = {"add", "subtract", "multiply"};
    
    for (int i = 0; i < 3; i++) {
        printf("Result of %s: %d\n", opNames[i], operations[i](a, b));
    }
    
    return 0;
}

Function pointers are widely used in callback mechanisms, event handling, and implementing polymorphism in C.

Variable Arguments

C allows functions to accept a variable number of arguments using the <stdarg.h> header.

#include <stdio.h>
#include <stdarg.h>

// Function with variable arguments
double average(int count, ...) {
    va_list args;
    double sum = 0;
    
    // Initialize the argument list
    va_start(args, count);
    
    // Access all arguments
    for (int i = 0; i < count; i++) {
        int num = va_arg(args, int);
        sum += num;
    }
    
    // Clean up
    va_end(args);
    
    return sum / count;
}

int main() {
    printf("Average of 2, 4, 6: %.2f\n", average(3, 2, 4, 6));
    printf("Average of 5, 10, 15, 20, 25: %.2f\n", average(5, 5, 10, 15, 20, 25));
    return 0;
}

When using variable arguments, you need some way to know how many arguments were passed or when to stop processing them. Common approaches include:

  • Passing the count as the first argument (as shown above)
  • Using a sentinel value (like NULL) to mark the end
  • Using a format string (like in printf) to determine the number and types of arguments
