C Keywords

This section tests linking of C keywords.

  • Storage classes: auto, extern, register, static
  • Type specifiers: char, double, float, int, long, short, void
  • Type qualifiers: const, volatile
  • Control flow: break, case, continue, default, do, else, for, goto, if, return, switch, while
  • Other keywords: enum, sizeof, struct, typedef, union

The sizeof operator is used to determine the size in bytes of a type or variable.

C Library Functions

This section tests linking of C library functions.

  • Memory management: malloc, free, calloc, realloc, memcpy, memset
  • String functions: strlen, strcpy, strcat, strcmp
  • I/O functions: printf, scanf, fprintf, fscanf, fopen, fclose, fread, fwrite
  • Error handling: perror, strerror, assert
  • Program control: exit

The malloc function is used to allocate memory on the heap.

C Preprocessor Directives

This section tests linking of C preprocessor directives.

  • File inclusion: #include
  • Macro definition: #define, #undef
  • Conditional compilation: #if, #ifdef, #ifndef, #else, #elif, #endif
  • Other directives: #pragma, #error, #line

The #include directive is used to include header files in your program.

C Programming Concepts

This section tests linking of general C programming concepts in regular text.

In C, a pointer is a variable that stores the memory address of another variable. Arrays are used to store multiple values of the same type in a contiguous block of memory. Strings in C are represented as arrays of characters terminated by a null character.

A structure is a user-defined data type that allows you to combine data items of different kinds. Functions are reusable blocks of code that perform a specific task. Parameters are variables in a function declaration, while arguments are the values passed to a function when it is called.

Memory management is crucial in C programming. A memory leak occurs when allocated memory is not freed, leading to resource exhaustion. A buffer overflow happens when a program writes data beyond the end of a buffer, potentially causing security vulnerabilities.

Error handling is important for robust C programs. Debugging techniques help identify and fix issues in your code.

Code Blocks

This section tests that terms inside code blocks are not linked.

#include <stdio.h>
#include <stdlib.h>

int main() {
    int* ptr = (int*)malloc(sizeof(int) * 10);
    
    if (ptr == NULL) {
        perror("Memory allocation failed");
        return 1;
    }
    
    for (int i = 0; i < 10; i++) {
        ptr[i] = i * 2;
    }
    
    for (int i = 0; i < 10; i++) {
        printf("%d ", ptr[i]);
    }
    printf("\n");
    
    free(ptr);
    return 0;
}

The code above demonstrates memory allocation, array manipulation, and proper cleanup.
